# 육아맘카페(외부 커뮤니티)로 ‘엔진’을 수익화하는 전략 (B2B2C)

## 왜 맘카페인가?
- 이미 “부모가 모여있는 트래픽 풀”이 존재
- 맘카페의 고질 문제: **정보 품질/중복 질문/광고/운영 부담**
- UJuz 엔진(입소 점수/TO 알림/AI/공구)을 붙이면:
  - 카페 운영자는 “유지/수익”을 얻고
  - 회원은 “문제 해결”을 얻고
  - UJuz는 “저CAC 유입 + 데이터 + 구독/수수료”를 얻음

---

## 1) 파트너 제품(카페 관리자에게 파는 것)
### P0: Widget Pack(무료~)
- 카페에 붙이는 위젯(링크/배너/게시판 상단 고정):
  - TO 알림 설정 링크
  - 입소 점수 1회 무료 체험 링크
  - 동네 추천 장소 TOP 카드
- 수익: 레퍼럴 기반 수익배분(성과형)로 시작 → 진입 장벽 최소

### P1: Cafe Pro(월 과금)
- 관리자 콘솔 + 운영 도구 묶음
  - 인기 질문/키워드 자동 요약(주간 리포트)
  - 반복 질문 FAQ 자동 생성 + 상단 공지 템플릿
  - 스팸/광고 의심 글 분류(알림)
  - “공동구매 수요” 자동 감지(게시글에서 수요 신호)
- 수익: 월 구독(예: 49,000~149,000원) + 성과형

### P2: GroupBuy Engine(수수료)
- 카페 전용 공동구매를 “엔진”으로 제공
  - 모집/참여/결제/정산/CS 템플릿 제공
  - 공동구매당 5~10% 수수료(또는 결제대행 수수료)
- 카페는 자체 공구를 더 쉽게 운영 → 수익/만족도 ↑

---

## 2) 수익 모델(3갈래) + 카페 인센티브 설계
### (A) 구독 레퍼럴 RevShare
- 카페 링크(코드)로 유입된 유저가 **N일 내 구독**하면
  - 카페에 첫 결제액의 10~30% 배분(예: 20%)
  - 리텐션에 따른 2~3개월 추가 배분 옵션(선택)

### (B) 커머스(공구) 수수료 공유
- UJuz 공구/딜을 카페에 공급하거나,
- 카페 공구를 UJuz가 엔진으로 운영해주고
  - 수수료의 일정 비율을 카페에 공유(예: 30~50% of fee)

### (C) 스폰서/로컬 광고(장기)
- 지역 시설(학원/키즈카페/영유/어린이집 컨설팅) 광고
- 단, “검증/표기/리뷰” 정책이 없으면 신뢰를 잃기 쉬움 → 늦게

---

## 3) 유입 구조(카페에서 UJuz로)
### 3-1. ‘문제 해결 링크’로 유입
- “대기/입소 점수 계산해보기”
- “TO 알림 받기”
- “우쥬봇 상담”
→ 단순 랜딩(웹) → 앱 설치/가입 → 커뮤니티/도구 사용

### 3-2. 카페 내 ‘미니앱’ 느낌(위젯)
- 게시판 상단에 “지금 TO 뜬 곳(요약)” 카드(단, 데이터 출처 합의 필수)
- 사용자 클릭 → UJuz로 이동

### 3-3. 카페 회원을 ‘딜/공구’로 연결
- 카페 운영자가 공구를 열면 UJuz 엔진으로 진행
- 카페 회원은 결제/배송/CS까지 일관 경험

---

## 4) 데이터 전략(카페 → 엔진) : “허가 + 합의 + 최소수집”
- **스크래핑/무단 수집은 금지** (법/플랫폼 리스크)
- 대신 2가지 안전한 경로:
  1) 카페 운영자가 제공하는 **공식 export/피드**(또는 제보 폼)
  2) 카페 내부에서 “TO 제보”를 받는 **파트너 전용 폼/봇**

이렇게 들어온 데이터는:
- TO 감지 → UJuz의 `to_alerts` 데이터로 변환
- 패턴(요일/시간/연령반) 집계 → 카페/유저에게 다시 제공

---

## 5) 초기 GTM (0→10개 카페) 실행
1) 카페 10곳 리스트업(핵심 지역/회원수/운영자 성향)
2) “무료 Widget + 성과형 정산” 제안으로 진입
3) 2주간:
   - 위젯 클릭/설치/구독 전환 데이터 공유
   - 카페 운영자에게 주간 리포트 제공(가치 증명)
4) 전환:
   - Pro(월 과금) 또는 공구 엔진 사용 계약

---

## 6) 성공 지표(파트너)
- 위젯 CTR
- 설치 전환(install / click)
- 구독 전환율(paid / install)
- 공구 GMV 및 수수료
- 카페 운영자 NPS(운영 부담 감소 체감)
